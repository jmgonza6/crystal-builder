Parameter file commands

Below is a complete listing of the available commands and their arguments when creating a parameter file. The order of appearance in the file is irrelevant.

Required

bravais:    define which class of Bravais lattice to build
syntax:  bravais      class

  • class = cubic or tetragonal or ortho or custom or library

bravais_style    define which Bravais lattice style to build
syntax:  bravais_style      style

  • style arguments:
    bcc or fcc or sc or diamond for cubic
    bcc or fcc or sc for tetragonal or ortho

run    define the behavior of how the application runs.
syntax:  run      style

  • style = build or render

save    write the data to the specified file
syntax:  save      format filename

  • format = dmol or lammps or vasp
  • filename = optional, used only if format = dmol or lammps

stoich    define the type and number of elements for the crystal
syntax:  stoich      type1:n1,type2:n2,type3:n3,...
examples: 

stoich    C:2
stoich    Na:1,Cl:1
stoich    C:10,H:16,N:8,O:24


Semi-required

begin_basis    define the basis coordinates of each atom in the crystal
syntax:
begin_basis
x1,y1,z1   /// atom 1
x2,y2,z2   /// atom 2
...
...
...
xn,yn,zn   /// atom N
end_basis
NOTE: Must be defined if bravais was defined to be custom. The coordinates must be listed in the stoichiometric order defined by stoich. Anything after the z coordinate is treated as a comment.

graphene_basis    define the number of basis atoms in a graphene lattice
syntax:  graphene_basis      atoms

  • atoms = 2 or 4
NOTE: Must be defined if use_prebuilt was defined to be graphene

use_prebuilt    select a pre-defined crystal from the library
syntax:  use_prebuilt      name

  • name = graphene or hBN or mdc-2h or mdc-1t or tmdc-2h or tmdc-1t or petn or tatb
NOTE: Must be defined if library was selected


Modifiers

data_modify    write the output data in fractional coordinates
syntax:  data_modify      fractional
NOTE: Only valid if writing in the VASP format.

lat_angles    define the angle between each lattice vector, in (˚)
syntax:  lat_angles      alpha=X,beta=Y,gamma=Z
NOTE: append, in any order 0 or more of these parameters. By default alpha = beta = gamma = 90˚.

lat_parameters    define the magnitude of each lattice vector, in (Å)
syntax:  lat_parameters      a=X,b=Y,c=Z
NOTE: append, in any order 0 or more of these parameters. By default a = b = c = 1.

super_cell    extend the lattice vectors of a single unit cell
syntax:  super_cell      nx=X,ny=Y,nz=Z
NOTE: append, in any order 0 or more of these parameters. By default nx = ny = nz = 1.
